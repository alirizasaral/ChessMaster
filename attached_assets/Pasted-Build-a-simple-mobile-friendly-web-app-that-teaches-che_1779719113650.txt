Build a simple mobile-friendly web app that teaches chess openings interactively.

## Goal

Create a minimal chess opening trainer. Users choose an opening lesson, play moves on an interactive chessboard, and receive coaching feedback from an LLM. Feedback is shown as chat and also read aloud using ElevenLabs.

No login. No server-side user persistence. Store all user progress in browser localStorage.

## Tech Stack

Use:
- React frontend
- Node/Express backend
- chess.js for chess rules and legal move validation
- A React chessboard library
- OpenAI API for LLM coaching
- ElevenLabs API for text-to-speech
- localStorage for progress

Use Replit Secrets for:
- `OPENAI_API_KEY`
- `ELEVENLABS_API_KEY`

Do not expose API keys in the browser.

## Core Screens

### 1. Progress Screen

Show a list of opening lessons.

Each lesson has:
- Opening name
- Short description
- Status: `not started`, `started`, or `finished`
- Button to start or continue

Users can jump to any lesson at any time.

Initial lessons:
- Italian Game
- Ruy Lopez
- Queen’s Gambit
- London System
- Sicilian Defense
- Caro-Kann Defense

### 2. Lesson Screen

Show:
- Lesson title
- Back button
- Undo button
- Finish lesson button
- Interactive chessboard
- Chat-style coaching panel
- Mute/unmute audio button

The layout must be mobile-friendly:
- Title/actions at top
- Chessboard in the middle
- Chat panel below
- Chat panel should be scrollable

## Chess Behavior

Use `chess.js` to:
- Validate legal moves
- Maintain board state
- Generate FEN
- Track move history
- Support undo

The user may make any legal move.

After every legal move:
1. Update the board
2. Save progress to localStorage
3. Send the current lesson, move, FEN, and move history to the backend
4. Backend calls OpenAI
5. Show LLM feedback in chat
6. Send the feedback text to ElevenLabs
7. Play the returned audio unless muted

If a move is illegal, reject it and show a short message.

## Local Storage

Persist all progress in browser localStorage.

Use a structure like:

{
  "lessons": {
    "italian-game": {
      "status": "started",
      "fen": "...",
      "moves": ["e4", "e5", "Nf3"],
      "chat": [],
      "lastUpdated": "..."
    }
  },
  "settings": {
    "sound": true
  }
}

Keep chat history reasonably short, for example the latest 20 messages per lesson.

## LLM Coaching

Create a backend endpoint:

`POST /api/coach`

Request body:
{
  "lessonId": "italian-game",
  "lessonName": "Italian Game",
  "fen": "...",
  "moves": ["e4", "e5", "Nf3"],
  "lastMove": "Nf3"
}

The LLM should act as a friendly chess opening coach.

Use this response structure every time:

Move feedback: Good / playable / risky / mistake

Pros:
- ...

Cons:
- ...

Better alternatives:
1. ...
2. ...

Next idea:
...

Rules for the LLM:
- Keep explanations short and beginner-friendly
- Always explain pros and cons
- Always suggest alternatives
- Do not be harsh
- Focus on opening principles
- Explain why a move fits or does not fit the opening
- Avoid long engine-style analysis
- Do not claim a move is illegal, because legality is handled by chess.js

## Text to Speech

Create a backend endpoint:

`POST /api/tts`

Request body:
{
  "text": "..."
}

Use ElevenLabs to generate speech from the coaching text.

Return playable audio to the frontend.

Add a mute/unmute toggle. If muted, do not request or play audio.

## UI Style

Keep the design simple and minimal.

Use:
- Clean cards
- Large touch targets
- Good spacing
- Mobile-first layout
- Neutral colors
- Clear status labels

Avoid:
- Login
- Accounts
- Leaderboards
- Complicated settings
- Engine evaluation bars
- Quizzes
- Heavy animations

## Completion Rules

A lesson can be marked finished when the user clicks “Finish lesson”.

When finished:
- Save status as `finished`
- Return to progress screen
- Show the lesson as finished
